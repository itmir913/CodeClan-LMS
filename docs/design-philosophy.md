# CodeClan LMS — 디자인 철학

## Vision

> **"코드에만 집중하게 하라"**
>
> 교사는 3번의 클릭으로 수업을 시작하고, 학생은 화면 밖으로 눈을 돌릴 필요가 없다.
> 구글 클래스룸이 해결하지 못한 정보 교과의 불편함을 제거한다.

---

## 4대 설계 원칙 (Design Pillars)

### Pillar 1 — Contextual Unity (맥락 통합)
학생은 LMS·IDE·컴파일러를 오가지 않는다. 문제 지문 + 코드 에디터 + 실행 결과가 한 화면에 존재한다. 교사는 과목을 선택하면 모든 도구가 그 컨텍스트 안에 있다.

### Pillar 2 — Effortless Authority (손쉬운 통제)
교사는 `과목 선택 → 분반 탭 → 세션 시작` 3번으로 수업을 시작한다. 수행평가 생성·배포도 3클릭 이내. 복잡한 설정은 템플릿으로 숨긴다.

### Pillar 3 — Instant Truth (즉각적 진실)
채점 결과는 텍스트보다 색·아이콘이 먼저다. ✅ 초록 = 통과, ❌ 빨강 = 오류+힌트. 학생의 진행률은 숫자가 아닌 시각적 진행 바로 표현한다.

### Pillar 4 — Growing Simplicity (자라는 단순함)
지금은 심플하게 시작한다. 구글 클래스룸을 완전히 대체할 기능(라인별 코드 리뷰, 제출 로그 시각화, wbox 자동채점)은 점진적으로 추가한다. 기능이 늘어도 기본 UX는 단순하게 유지한다.

- **UI 폰트**: Pretendard → -apple-system → system-ui (한국어 최적화)

## UX 8원칙

1. **사이드바 = 과목 목록만** — 모든 기능은 과목 선택 후 컨텍스트 안에서 접근. 사이드바에 도구 메뉴 없음.

2. **분반 탭은 항상 상단에** — "지금 어느 반을 보고 있는가"를 항상 명시. 과목 선택 후 상단에 내 분반 탭 고정.

3. **3-Click Rule** — 교사가 세션 시작까지 3번 이내. 과목(1) → 세션 탭(2) → 시작 버튼(3). 이 이상이면 설계 실패.

4. **학생 코드 화면은 좌우 분할 고정** — 왼쪽 문제 지문(40%) + 오른쪽 에디터·결과(60%). 학생이 다른 프로그램을 켤 이유가 없어야 한다.

5. **채점 결과는 색·아이콘 먼저** — ✅/❌/🟡가 먼저, 텍스트 설명은 2차 정보. 인지 부하 최소화.

6. **진행 중인 세션은 최상단 배너 고정** — 어느 화면에 있든 세션이 진행 중이면 최상단 붉은 배너로 표시.

7. **빈 상태는 다음 행동 안내** — "없습니다" 대신 "아직 없어요 — [+ 추가]" 형태로 CTA 항상 포함.

8. **Progressive Disclosure** — 고급 기능은 접힌 섹션·설정 탭으로 감춤. 기본 UX는 항상 단순하게.

## Google Classroom 대비 차별점

| 항목 | Google Classroom | CodeClan LMS |
|---|---|---|
| 코드 제출 | 파일 첨부 또는 링크 | 웹 내장 에디터에서 바로 작성·제출 |
| 코드 채점 | 교사가 파일 다운·실행 | 자동 채점 (wbox test case) |
| 피드백 | 줄글 댓글 | 채점 결과 즉시 시각화 + 인라인 힌트 |
| 진도 관리 | 제출 O/X만 | 코드 작성 로그, 진행률 시각화 |
| 과목 구조 | Class = 과목+분반 혼합 | 과목 → 분반 명확히 분리 |

## 개발 우선순위

1. **현재 (MVP)**: 과목 중심 IA + 교사 3-click 워크플로우 + 기본 채점
2. **다음 단계**: wbox 자동채점 연동 + 학생 코드 에디터 분할 화면
3. **장기**: 라인별 코드 리뷰, 제출 로그 시각화, Google Classroom 완전 대체
